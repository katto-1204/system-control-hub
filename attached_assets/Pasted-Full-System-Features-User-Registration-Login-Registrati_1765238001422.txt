Full System Features
User Registration & Login:


Registration: New users (students, faculty) can create accounts using their email addresses, student IDs, and roles (Student or Faculty).


Login: Secure login with username/email and password. Role-based access control (students can only request, admins can approve or reject).


Password Reset: Users can reset passwords via email.


Dashboard:


Displays available facilities, equipment, and events.


Users can view their upcoming bookings.


Admins can view a summary of all upcoming events and facilities.


Facility Availability Checker:


Real-time facility and equipment availability checker.


Users can filter based on date, time, and type of facility.


Event Request Submission:


Users can submit event requests with required details like event name, purpose, date, time, requested facility, and equipment needed.


Users can upload supporting documents (e.g., event plans).


Admins can view and process these requests.


Notification System:


Users receive notifications when their event request is approved or rejected.


Email and system notifications for event status changes (pending, approved, rejected).


Admin Approval Panel:


Admins can view all pending, approved, and rejected event requests.


Admins can approve or reject requests.


Admins can monitor event progress, assign staff for event support, and access a full event history.


Real-Time Calendar:


A shared calendar showing booked and available facilities.


Admins and users can see facility schedules in real-time.


Time blocks for each facility and equipment are color-coded (available, booked, pending).


Reporting & Analytics:


Admins can generate reports on facility usage, equipment demand, and event frequencies.


Reports can be exported as CSV or PDF.


User Profile Management:


Users can update personal details and view their event booking history.


Admins have access to modify any user details or reset passwords.


Admin Management:


Manage user roles (assign, edit, remove).


View user activity and track event requests over time.

SQL Schema with Transactions, Functions, and Stored Procedures
Database Tables:
Users Table:

 CREATE TABLE users (
  user_id INT AUTO_INCREMENT PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,
  first_name VARCHAR(100),
  last_name VARCHAR(100),
  role ENUM('student', 'faculty', 'admin') NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);


Facilities Table:

 CREATE TABLE facilities (
  facility_id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  description TEXT,
  capacity INT,
  status ENUM('available', 'booked') DEFAULT 'available',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);


Bookings Table:

 CREATE TABLE bookings (
  booking_id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT,
  facility_id INT,
  event_name VARCHAR(255),
  event_date DATE,
  start_time TIME,
  end_time TIME,
  equipment_needed TEXT,
  status ENUM('pending', 'approved', 'rejected') DEFAULT 'pending',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(user_id),
  FOREIGN KEY (facility_id) REFERENCES facilities(facility_id)
);


Notifications Table:

 CREATE TABLE notifications (
  notification_id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT,
  message TEXT,
  status ENUM('unread', 'read') DEFAULT 'unread',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(user_id)
);


Stored Procedures & Functions:
Approve Event Request:

 DELIMITER //
CREATE PROCEDURE approve_event(booking_id INT)
BEGIN
  UPDATE bookings
  SET status = 'approved', updated_at = CURRENT_TIMESTAMP
  WHERE booking_id = booking_id;

  UPDATE facilities
  SET status = 'booked'
  WHERE facility_id = (SELECT facility_id FROM bookings WHERE booking_id = booking_id);
END //
DELIMITER ;


Reject Event Request:

 DELIMITER //
CREATE PROCEDURE reject_event(booking_id INT)
BEGIN
  UPDATE bookings
  SET status = 'rejected', updated_at = CURRENT_TIMESTAMP
  WHERE booking_id = booking_id;
END //
DELIMITER ;


Get Available Facilities:

 CREATE FUNCTION get_available_facilities(event_date DATE, start_time TIME, end_time TIME)
RETURNS TABLE
BEGIN
  RETURN
  (SELECT * FROM facilities WHERE status = 'available' 
   AND NOT EXISTS (
      SELECT 1 FROM bookings 
      WHERE facility_id = facilities.facility_id
      AND event_date = event_date
      AND ((start_time BETWEEN start_time AND end_time) OR (end_time BETWEEN start_time AND end_time))
   ));
END;
